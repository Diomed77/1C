Настройка параметров для мобильного приложения.
         
Данные набор настроек предназначен для удобства настройки обмена данными между 
мобильными приложениями, установленных в мобильных устройствах агентов и
1С: Предприятием, а также для дополнительных настроек для работы с модулем обмена.  

Поддерживается возможность настройки для 2-х видов мобильных приложений, 
используемых в мобильных устройствах агентов:

	1. Агент+ версии 1.5                                                    
	2. Агент Плюс: Мобильная торговля

Часть настроек для этих приложений пересекаются.
Для переключения настроек, доступных для того или иного приложения используется 
соответствующие пункты меню "Переключить список настроек", находящегося на верхней 
панели управления формой. Т.о. настройки, доступные для определенного вида приложения
будут отображаться или скрываться.    

Настройки на форме логически сгруппированы на 2-х закладках. Не все константы являются
обязательными к заполнению. 

Общие настройки                  

• Название БД в МУ - название базы данных для мобильного приложения. По умолчанию
  проставляется название "Основная". Не нужно изменять значение данной коснтанты
  без необходимости. Изменять название БД следует только в том случае, если в МУ
  используется несколько баз данных.

• Группа новых клиентов - группа справочника «Контрагенты», в которую загружаются 
  созданные в МУ новые клиенты.

• Режим загрузки заявок - определяет действие при загрузки документа "Заявка" из МУ 
  в ЦБД.                                                     

• Вид операции для заявок - указывается вид операции, который будет проставляться по 
  умолчанию для загружаемых с МУ документов "Заявки". Причем, если указан вид операции
  "Заявка на склад",  то в качестве склада в документе будет проставляться склад, указанный 
  в качестве реквизита "Основной склад" в справочнике "Агенты".

• Режим загрузки документов поступления - выбирается способ загрузки документа 
  «Поступления» с МУ:
	1. Как документ «Возврат от покупателя» – в ЦБД будет создан документ «Возврат от 
	    покупателя»;
	2. Как документ «Поступление ТМЦ» – в ЦБД будет создан документ «Поступление ТМЦ».

• Папка фотографий документов - каталог в МУ, в которой будут храниться фотографии к 
  документам, сделанными агентов в мобильном приложении.

• Папка картинок товаров - каталог в МУ, в котором хранятся картинки товаров, сделанные
  агентом в в мобильном приложении.       
  
 • Минимальный тип цены  - минимально допустимый тип цены для товаров в МУ. Если указано
   данное значение, то в МУ нельзя будет оформлять документы продаж для номенклатуры,  у 
   которой значение цены по выбранному в МУ типу цены ниже, чем значение цены по указанному 
   в этой константе типу цену.
  
• Срок перемещений  - период в днях, за который будут выгружаться документы "Перемещения 
  номенклатуры". Если не указано значение константы, то перемещения в МУ не будут
  выгружены.                 
  
• Срок инвентаризаций  - период в днях, за который будут выгружаться документы "Инвентаризация 
  ТМЦ" в МУ. Если не указано значение константы, то инвентаризации в МУ не будут
  выгружены.
  
• Интервал заявок - интервал выгрузки документов "Заявка" в мобильные устройства (дней).
  Если указано значение, больше 0, то в МУ агентов будут выгружаться документы "Заявки" за 
  указанное коли-во дней, отсчитывая от текущей даты. Данная константа используется только
  для при использовании приложения "Агент Плюс: Мобильная торговля".

• Интервал подтверждений - интервал в днях, за который будут выгружаться подтверждения 
  для документов в МУ агентов; 

• Дата истории продаж -  дата, по которую нужно выгружать историю продаж из ЦБД в МУ. Если 
  значение не указано, то история продаж выгружается по точку актуальности итогов.
                             
• Использовать планы продаж. Если установлен флажок, то в МУ производится выгрузка
  план-фактных данных по продажам в МУ.
   
• Режим планирования продаж - режим формирования документа "План продаж" по умолчанию. 

Настройки для шифрования файлов обмена: 

• Шифровать данные - включает / выключает шифрование данных. 

• Крипто-пароль - пароль для дополнительного шифрования файлов.

                                                                                                                                                        
Настройки для обмена данными:

• Способ обмена - способ обмена данными, используемый по умолчани. В зависимости от 
  выбранного значения, становятся доступными соответствующие настройки для выбранного
  способа обмена:    
  
  Константы для обмена с помощью Агент+ Сервера
	• Адрес сервера - адрес компьютера, на котором запущена служба "Агент Плюс СОД". В 
	  качестве адреса сервера указывается либо сетевое имя компьютера или его ip-адрес в 
	  локальной сети и через двоеточие указывается порт, через который работает служба 
	  "Агент Плюс СОД". Пример: aplussrv:5555, 192.168.0.1:5555.
	• Папка обмена - директория для обмена данными с помощью "Агент Плюс СОД". Данная
	  папка должна совпадать с каталогом, указанной с настройках службы "Агент Плюс СОД".

  Константы для обмена с помощью FTP-cервера
	• Адрес сервера - адрес FTP-Сервера, через который производится обмена данными.
	• Имя пользователя - имя пользователя, с помощью которого производится авторизация
	  на FTP-сервере.
	• Пароль - пароль для пользователя, указанного в костанте "Пользователь".

	
• Использовать единицы измерения - Режим использования единиц измерения в табличной 
  части документов в МУ. Если данная константа не включена, то количество выбранного 
  товара всегда будет пересчитываться в базовые единицы измерения. Если же она включена,
  то формирование документов будет возможным осуществлять в указанных единицах измерения.
  
• Использовать торговые точки - если указан флажок, то учет будет вестись с использованием
  торговых точек, подчиненных контрагенту. Иначе, работа с контрагентами будет вестись без
  использования торговых точек (при этом никакие данные, связанные с использованием 
  торговых точек, при обмене данными выгружаться и загружаться не будут).         
        
• Использовать договоры - признак использования договоров в МУ. Если установлена данная 
  константа, то в МУ включается режим работы с договорами (на форме документов становится 
  доступным поле "Договор"). Соответственно, при загрузке документов с МУ, в реквизит "Договор"
  в загружаемых документах будет проставлен тот договор, который был выбран агентом в МУ.
     
• Использовать время в маршрутах - использовать время желательного посещения при 
  формировании документов "Маршруты".

• Использовать персональные цены - если выбран этот флажок, то для клиентов будут
  выгружаться персональные цены при условии, если в справочнике "Контрагенты (для мобильных
  устройств)" для соответствующего клиента выбран флажок "Персональные цены". 

• Использовать автонумерацию документов в ЦБД - если выбран флажок, то при загрузке 
  документов из МУ номер будет присвоен системой автоматически. В противном случае 
  у документов будут проставлены те номера, которые присвоены им в МУ.
    
Настройки для работы с картами    

• Использовать GPS в мобильных устройствах - признак использования GPS в мобильных 
  устройствах агентов.

• Приложение для просмотра карт - выбирается тип используемого приложения для просмотра
  карт. Может быть выбрано либо использование стандартной утилиты компоненты APPlus.dll, 
  либо использование внешнего приложения Map9 Viewer. 

• Основная карта - основная карта для отображения маршрутов агентов (агента). Используется
  как карта по умолчанию, которая выбирается в отчетах "Карта маршрута агента" и "Контроль
  передвижений агентов".
  
• Погрешность при расчете факта посещения точки - погрешность в метрах, которая влияет на
  расчет факта посещения торговой точки/клиента при формировании отчета "Карта маршрута 
  агента" в виде расшифровки.

• Путь к файлу приложения "Map9 Viewer" - указывается путь к файлу запуска приложения "Map9
  Viewer" (необходима при формировании отчета "Контроль передвижений агентов"). 

• Режим загрузки треков - выбирается способ загрузки треков перемещения агентов из МУ: 

     - Загружать каждую точку трека в ЦБД;

     - Загружать файл треков в ЦБД целиком (только при данном способе загрузки важно
	 указание константы, которая описана ниже).

• Папка хранения треков - указывается каталог, предназначенный для хранения файлов с
  треками перемещений агентов, которые загружаются из МУ.
